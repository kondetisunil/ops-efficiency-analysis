Marketplace Efficiency Analysis
This project demonstrates the application of SQL to solve operational bottlenecks in a three-sided marketplace (Customer-Merchant-Courier).

Business Problem
In a rapidly expanding market like Germany, maintaining the balance between courier supply and customer demand is critical for preventing churn.

Logic Applied
Zone Bottlenecks: Identified specific city zones where the average delivery delay exceeded 10 minutes.

Peak Hour Optimization: Analyzed hourly order volume against courier payouts to identify "under-supplied" windows.

Note: This is a logic-based project using hypothetical datasets to demonstrate strategic data extraction.
